# 10 - ロードマップ

## MVP の定義

「最小限で楽しい」を目指す。以下が動けば MVP とみなす。

### MVP に含むもの

- ATP OAuth ログイン
- 自分のタイムライン表示
- 投稿の作成 (テキストのみ)
- いいね、リプ、引用リポスト、リポスト
- Browser LLM による投稿タグ付け (質問、感情、ユーモア、分析、意見、共感系の識別)
- 気質診断 (認知機能プロトタイプ類似度)
- 5 軸ステータスの計算と表示 (% のみ)
- 16 ジョブのマッチングと表示 (ユーザー選択の名前バリアント)
- 毎日のクエスト自動生成 (3 個、機械的)
- クエスト進捗のリアルタイム更新
- タイムラインバッジ (クエスト関連のみ)
- 精霊の定型セリフ表示
- 目標ジョブの設定
- 他ユーザーとの共鳴度計算 (プロフィール画面で表示)
- **共鳴タイムライン (目玉機能)**: 発見ディレクトリに登録されたユーザーの中から、共鳴度の高い相手の投稿を時系列で流す (05-compatibility.md §共鳴タイムライン)
- 発見ディレクトリの手動追加運用 (管理ダッシュボード経由、14-admin.md §(e))
- 発見オプトインの設定 UI (profile.discoverable)
- 旅の仲間リスト (最大 10 人)
- PDS へのデータ保存
- 診断結果の公開/非公開設定
- 設定画面 (基本項目のみ)
- レスポンシブ (モバイル縦長最適化)
- ダークモード

### MVP に含まないもの (v1.1 以降)

- 外部 LLM による精霊会話 (BYOK 機能、Anthropic / OpenRouter)
- 外部 LLM による投稿ドラフト生成 (BYOK)
- 外部 LLM による上位診断 (BYOK)
- 画像・動画の投稿とリッチメディア表示
- 検索機能
- フィード (Following 以外の Discover など)
- 通知の細かい設定
- 多言語対応
- 共鳴マップ (シェア用画像)
- ペアジョブ称号
- 称号システム全般
- コンボ倍率の可視化
- ステータスの時系列グラフ
- React Native ネイティブアプリ

## リスクと対応

### WebGPU 非対応ブラウザのユーザー体験低下

**症状**: WASM フォールバックで診断に 30 秒以上かかる

**対応**:
- 初回起動時に対応状況を検知し、「この環境では処理が遅くなります」と事前告知
- 該当ユーザーには BYOK モードで Claude API 利用を推奨
- 対応ブラウザの推奨 (Chrome, Edge)

### モデルダウンロードの 130MB が重い

**症状**: モバイル回線で初回起動が遅い、データ消費を嫌がるユーザー

**対応**:
- Wi-Fi 環境での初回起動を推奨
- 「一度だけの儀式」と明確に伝える
- IndexedDB キャッシュで 2 回目以降は即座
- 将来的に q4 量子化など軽量版への切り替えオプション

### PDS のレート制限超過

**症状**: ヘビーユーザーが 1 日に大量書き込みをすると弾かれる

**対応**:
- 書き込みは日次バッチ化
- IndexedDB を先にコミット、PDS は遅延同期
- 失敗時はリトライキューに入れる

### 診断結果への違和感によるユーザー離脱

**症状**: 「別の診断サイトでは『冒険者』寄りだったのに、ここでは『賢者』と言われた」

**対応**:
- 信頼度表示を明示 (「投稿数が少ないので揺らぎます」)
- 認知機能スコアと根拠投稿を見せる詳細画面
- ユーザーが自認ジョブをフィードバックできる UI
- 時間経過で診断がブレることを前提にした UX

### 商標への対応不足

**症状**: 商標権者からの警告

**対応**:
- UI・ソース・ドキュメント・データファイルから商標性のある 4 文字コード体系と商標名を完全排除済み
- プライバシーポリシーと About で「ユング派の認知機能に基づく独自のジョブ体系」と明記
- 警告を受けたら即座に残存表記を除去

## オープンソース戦略

**検討中**: コア部分 (`packages/core`, `packages/lexicons`) を MIT ライセンスで公開するかどうか。

**公開のメリット**:
- AT Protocol コミュニティでの認知向上
- 他開発者がレキシコンを参照できる → 相互運用性
- コードレビューの機会

**公開のデメリット**:
- プロトタイプ投稿集の盗用リスク
- クローンアプリが出る可能性

**結論 (暫定)**: レキシコンとジョブ配分は公開、プロトタイプとクエストテンプレートは非公開、が妥当。
