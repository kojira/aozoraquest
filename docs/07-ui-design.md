# 07 - UI 設計

## デザイン原則

### 1. 変化する情報が画面スペースを取る

静的な情報 (ジョブ名、気質の分類、ステータスの絶対値) は常時大きく表示しない。毎回開いても変わらない情報は、ユーザーにとって徐々に情報量がゼロに近づく。

逆に、1 回の行動で変わる情報 (クエスト進捗、獲得 EXP、コンボ数) を前面に出す。これによりアプリを開く価値が持続する。

### 2. モバイル縦長優先

スマホの縦長レイアウトを基準にデザインする。タブレット・PC はサイドバー付きレイアウトに拡張。

- Phone: 380px 幅想定 (Safe area 含む)
- Tablet: 768px 以上でサイドバー開始
- Desktop: 1024px 以上で 3 カラム

### 3. 情報密度の勾配

上から下にかけて、**固定情報 → 動的情報 → メインコンテンツ** の順に並べる。

```
[アイデンティティ] ← ほぼ変わらない
[今日の活動]        ← 動的、圧縮表示
[メインタイムライン] ← 主役
```

### 4. アコーディオン推奨

詳細情報はアコーディオンで畳む。閉じた時はアイコン + 数値サマリー。開いた時に全情報を見せる。

## タイムラインタブ

Home のタイムラインは **フォロー / 共鳴** の 2 タブで構成する。切替バーは sticky ヘッダーの直下 32px。

### フォロー TL (既定)

標準的な Bluesky のフォローベースタイムライン。`app.bsky.feed.getTimeline` をそのまま表示。

### 共鳴 TL (目玉機能)

フォロー外でも、Aozora Quest の発見ディレクトリに登録された他ユーザーの中から、自分と共鳴度の高い相手の投稿を集めて時系列で流す (05-compatibility.md §共鳴タイムライン)。

各投稿に、通常のバッジに加えて **著者の共鳴度バッジ** を右上に追加:

```
[ava] Kaori @kaori.bsky 24m  [共鳴 72%]  ♥+1 LUK
雨上がりの夕方、通学路のアスファルトが…
```

共鳴度バッジは著者のジョブカラー (03-game-design.md §色と配置) のピル形、横幅約 64px。タップすると相手のプロフィールに遷移し、詳細な共鳴度が見られる。

### 空状態 / ロード中

- ディレクトリに自分以外の登録者がいない場合: 「旅人たちがまだ集まっていません。最初の一人になって、仲間を待ちましょう。」 + [発見 ON にする] ボタン
- スコア計算中: プログレスバー + 精霊の一言「共鳴する声を探している…」

### 発見オプトインの案内

共鳴 TL タブを初めて開いたとき、自分が未オプトインなら控えめなバナー: 「あなたの投稿も共鳴 TL に流したい？ 設定で『発見 ON』」。閉じるとしばらく再表示しない (7 日クールダウン)。

## タイムラインバッジ

### 位置とサイズ

投稿の右上、時刻表示の横。横幅約 70px、高さ 18px。

```
Kota @kota.bsky 12m         ♥+1 LUK
                              ~~~~~~
                              バッジ
```

### バッジのバリアント

| アイコン | アクション | 表示例 |
|---|---|---|
| ♥ | いいね贈与 | `♥+1 LUK` |
| ↩ | リプ | `↩+1 AGI` |
| 📎 | 引用リポスト | `📎+3 INT` |
| ↻ | リポスト | `↻+1 AGI` |

アイコンは絵文字ではなく SVG で実装 (アクセシビリティ、統一感)。

### 色

該当ステータスの色ランプの `200` 背景 + `800` テキスト。視認性は保ちつつ控えめに。

```css
.badge-atk  { background: #FAECE7; color: #712B13; }
.badge-def  { background: #E6F1FB; color: #0C447C; }
.badge-agi  { background: #FAEEDA; color: #633806; }
.badge-int  { background: #EEEDFE; color: #3C3489; }
.badge-luk  { background: #FBEAF0; color: #72243E; }
```

### レーダーチャートの描画

5 軸の正五角形。

- 外周 = 100% 相当 (全軸の最大可能値)
- 各軸の点 = 正規化した値
- 現在: 紫の半透明塗り
- 目標: 緑の破線 (オーバーレイ)
- 軸ラベル: ATK / DEF / AGI / INT / LUK (頭文字のみ、フルネームは下の一覧で)

SVG で実装。`react-native-svg` とも互換性があるようにする。

## 設定画面

### セクション構成

1. **プロフィール**
   - ジョブ表示名のバリアント (既定 / 生産系 / 別系統)
   - 診断結果の公開設定 (publicAnalysis)
   - 発見 ON/OFF (discoverable、共鳴 TL への掲載オプトイン。ON にすると publicAnalysis も自動的に ON)
2. **AI 接続 (BYOK)**
   - プロバイダー選択: Anthropic / OpenRouter
   - API キーの入力 (暗号化して IndexedDB 保存)
   - モデル選択 (プロバイダーごとに選択肢が変わる。Anthropic: Haiku / Sonnet / Opus、OpenRouter: 一覧から検索)
   - 接続確認ボタン (小さな test プロンプトで疎通確認)
   - キー未設定の場合、LLM 依存機能 (自由対話、上位診断、投稿ドラフト) はグレーアウト
3. **通知**
   - クエスト到着通知
   - 仲間追加通知
   - Bluesky 標準通知
4. **表示**
   - テーマ (ライト / ダーク / システム)
   - フォントサイズ
   - アニメーション (控えめ / 標準 / リッチ)
5. **データ**
   - 診断結果の再実行
   - キャッシュクリア
   - データのエクスポート (JSON)
6. **アカウント**
   - ログアウト
   - 別アカウントでログイン
7. **アプリ情報**
   - バージョン
   - ライセンス
   - プライバシーポリシー
   - 利用規約

## ダークモード

全画面でライト / ダークモード対応。基本方針:

- 純黒ではなく `#0A0A0D` 程度のオフブラック
- ステータス色は明度を上げる (ダーク背景で暗すぎないよう)
- 画像は sepia 10% で目に優しく

CSS カスタムプロパティで色を管理し、`prefers-color-scheme` メディアクエリで切り替え。

## アクセシビリティ

- すべてのインタラクティブ要素は 44x44px 以上 (iOS ガイドライン)
- フォントサイズはユーザー設定を尊重 (`rem` 単位)
- コントラスト比 WCAG AA 以上
- 色だけでなく形や文字でも情報を伝える
- スクリーンリーダー対応 (`aria-label` 等)
- `prefers-reduced-motion` でアニメーション抑制

## アニメーション原則

- 基本は抑制気味。派手さより穏やかさ
- `prefers-reduced-motion: reduce` でほぼ全停止
- 使う場面:
  - クエスト達成時の進捗バー伸び
  - LV アップ時の光沢
  - 転職時の特別演出 (ここだけ派手でも良い)
  - ローディング (精霊召喚の呼吸)
- 使わない場面:
  - タイムラインのスクロール (滑らか以上の演出不要)
  - バッジ表示 (瞬時でよい)

## コンポーネントライブラリ

- React + Tailwind CSS
- shadcn/ui をベースに、カスタムコンポーネントを構築
- アイコンは Lucide React
- アニメーションは Framer Motion (必要箇所のみ)
- レーダーチャート、折れ線グラフ等は Recharts または自前 SVG
